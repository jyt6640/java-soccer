# 🧩 미션 제목
축구 경기 결과 분석기


# 🔍 진행 방식
미션은 과제 진행 요구 사항, 기능 요구 사항, 프로그래밍 요구 사항의 세 가지로 구성된다. 기능 구현 전에 우선 "기능 목록"을 작성하고, 기능 단위로 커밋한다. 기능 요구 사항에 명시되지 않은 부분은 스스로 판단하여 구현하도록 안내한다.


# 🚦 과제 제출 전 체크 리스트
- 요구 사항 명시, 출력/입력 형식 미준수 시 0점 처리
- 모든 테스트 성공 여부 확인
- Java 21 사용 (java -version으로 확인)
- Mac/Linux: ./gradlew clean test
  Windows: gradlew.bat clean test 또는 .\\gradlew.bat clean test


# 🚀 기능 요구 사항
## 문제 설명
축구 경기에서 각 팀의 점수를 입력받아, 승패 여부와 최종 결과를 출력하는 프로그램을 작성한다. 두 팀의 점수와 경기 종료 여부를 입력받는다.


## 입력 예시
```
입력
팀A 3
팀B 2
종료
```


## 출력 예시
```
출력
팀A 승리
```


## 기능 목록
1. 팀 이름과 점수를 입력받아 저장하기
2. 경기 종료 여부 확인 및 결과 계산 로직 구현
3. 승패 결과에 따라 적절한 메시지 출력하기
4. 잘못된 입력(예: 점수 음수) 시 오류 메시지 출력하기
5. 프로그램 종료 조건 처리 (종료 키워드 입력 시)


## 🚀 상세 기능 목록

기능 목록은 개발 과정에서 지속적으로 업데이트될 수 있습니다.

### 1. 입력 기능
- [ ] 게임 진행을 위한 입력을 받는다.
    - **팀 이름**과 **해당 팀의 점수**를 입력받는다. (예: `팀A 3`)
    - **게임 종료 여부**를 입력받는다. (예: `종료`)
- [ ] 입력값은 쉼표, 공백 등 지정된 구분자를 기준으로 파싱한다.

### 2. 게임 로직 (핵심 도메인)
- [ ] **점수 비교 및 승패 판단**
    - 두 팀의 점수를 비교하여 승리, 무승부, 패배를 판단한다.
    - 예: A팀 3점, B팀 2점 -> A팀 승리
- [ ] **경기 기록 생성**
    - 각 경기의 결과(팀 이름, 점수, 승패)를 생성하고 관리한다.

### 3. 출력 기능
- [ ] 경기 결과를 사용자에게 출력한다.
    - 승리한 팀, 혹은 무승부 결과를 포맷에 맞춰 출력한다.
- [ ] 예외 발생 시 에러 문구(`[ERROR]`)를 출력한다.

### 4. 예외 처리 (Validation)
정상적인 경우뿐만 아니라 발생할 수 있는 예외 상황을 사전에 방지한다.

- [ ] **팀 이름 유효성 검사**
    - 팀 이름이 비어있거나 공백인 경우
- [ ] **점수 유효성 검사**
    - 점수가 숫자가 아닌 경우
    - 점수가 음수인 경우
- [ ] **입력 형식 검사**
    - 입력 형식이 지정된 포맷과 다른 경우
    - 종료 키워드 외에 알 수 없는 명령어가 입력된 경우
